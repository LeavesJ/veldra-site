Policy

Reason codes and context

A reject is credible only when it names the failure and shows the numbers. ReserveGrid OS emits stable codes plus policy context.

Reason codes

Stable names, stable meaning

Code Category Description
AverageFeeTooLow fees Average fee per transaction (sats per tx) is below the effective minimum for the selected fee tier.
TotalFeesTooLow fees Total fees implied by the template are below min_total_fees.
CoinbaseZero template coinbase_value is zero and reject_coinbase_zero is enabled (applies to non empty templates).
TooManyTransactions limits Template tx_count exceeds max_tx_count.
PrevHashWrongLen template prev_hash length does not match required_prevhash_len.
UnsupportedVersion policy Template protocol version does not match protocol_version.
EmptyTemplate template tx_count == 0 and reject_empty_templates is enabled.

Sample policy.toml

Readable and diffable

# Example only. 

[policy]
protocol_version = 1
required_prevhash_len = 64

min_total_fees = 0
max_tx_count = 100
min_avg_fee = 0

low_mempool_tx = 50
high_mempool_tx = 500

# Required by current PolicyConfig 
tx_count_mid_threshold = 10
tx_count_hi_threshold = 50

# Dynamic tier floors (avg fee per tx, sats/tx)
min_avg_fee_lo = 0
min_avg_fee_mid = 0
min_avg_fee_hi = 0

max_weight_ratio = 0.999

reject_empty_templates = true
reject_coinbase_zero = false
unknown_mempool_as_high = true
Tier input
mempool tx count (or unknown fallback)
Tier output
low | mid | high

scope and safety

Security boundaries

Veldra is infrastructure code. Treat the demo as local only unless you add authentication and network hardening.

What this is

Policy evaluation and observability for templates. It produces verdicts with reason codes and policy context.

  • Verifier and dashboard
  • Policy driven thresholds
  • Deterministic logging

What this is not

No custody, no wallet logic, no key management. It is not a mining pool and not a payment system.

  • No private keys
  • No signing authority
  • No exchange integration

Hard rule

Do not expose the dashboard to the public internet without authentication. Treat all endpoints as sensitive operational surfaces.

If you need a public demo, use a screen recording or a protected tunnel with authentication.

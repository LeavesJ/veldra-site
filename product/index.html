ReserveGrid OS

Template verification built for production

Evaluate block templates against operator policy with explainable rejections, live observability, and structured exports. Built in Rust for Bitcoin mining pools.

Read the Docs View on GitHub

How it works

Templates in, verdicts out. One contract, end to end.

01

Template source

Stratum v2 bridge or bitcoind produces a candidate block template.

→
02

Template manager

Packages the candidate as a TemplatePropose message.

→
03

Policy verifier

Evaluates against policy.toml, emits TemplateVerdict with context.

→
04

Dashboard + logs

Live verdicts, aggregates, CSV/NDJSON export.

Capabilities

Core capabilities

✓

Explainable rejects

Every rejection includes reason_code and policy_context, so you understand exactly why a template was rejected.

◆

Mempool-aware fee tiers

Automatic low, mid, and high fee tier classification based on real-time mempool transaction counts.

▦

Live operator dashboard

Real-time verdicts, aggregates, and charts for instant visibility into template acceptance rates.

⚙

Policy wizard

Generate policy.toml from a simple UI instead of hand-editing TOML configuration files.

≡

Deterministic evaluation

Same template and policy always produce the same verdict. No randomness, 100% reproducible.

■

Structured exports

NDJSON logs, CSV reports, and stats API for easy integration with monitoring and compliance tools.
